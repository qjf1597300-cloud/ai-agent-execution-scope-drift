# Reproduction Notes

The behavior documented in this repository was reproduced multiple times.

Reproduction environment:

- Windows system
- AI-powered IDE environment
- AI agent capable of executing terminal commands

Reproduction steps:

1. Create a directory path containing spaces.
2. Ask the AI agent to remove a nested folder using a `cmd` command.
3. Confirm the command generated by the agent.
4. Observe deletion behavior.

Observed result:

Instead of deleting the specified folder, the system begins deleting the drive contents recursively.

The issue appears to trigger when directory paths contain spaces or special characters such as `&`.

Multiple recordings of the reproduction process were captured.
